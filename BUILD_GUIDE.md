# Сборка и запуск обработки — пошагово

Идём от «платформа установлена» до «`.epf` работает и совпадает с примерами».
UI-термины даны как в русском Конфигураторе. Объекты и реквизиты нужно называть
**ровно** так, как здесь — на эти имена ссылается код.

---

## Фаза 0. Пустая информационная база

1. В окне запуска 1С → **Добавить** → «Создание новой информационной базы» → Далее.
2. «Создание информационной базы **без конфигурации** для разработки новой
   конфигурации…» → Далее.
3. Имя базы: `Ведомости_Тест` → «На данном компьютере…» → путь (любая пустая папка) → Готово.
4. Выделить базу → **Конфигуратор**.

## Фаза 0.1. Свойства конфигурации

1. Меню **Конфигурация → Открыть конфигурацию**.
2. Правой кнопкой на корневом узле «Конфигурация» → **Свойства**:
   - «Основной режим запуска» = **Управляемое приложение**;
   - «Режим совместимости» = **Не использовать** (нативный режим 8.3.25+).

## Фаза 1. Метаданные

### 1.1 Перечисление `СтатусыДокументов`
Правой кнопкой на «Перечисления» → Добавить. Имя = `СтатусыДокументов`.
Вкладка **Данные → Значения перечисления**, добавить по порядку:
`Создан`, `Внутренний`, `Акцептован`, `Заблокирован`.

### 1.2 Справочник `Договоры`
Добавить справочник, Имя = `Договоры`.
- Вкладка **Данные**: «Длина наименования» = **100**.
- **Реквизиты** (кнопка Добавить):
  - `Активен` — тип **Булево**;
  - `НачалоДействия` — тип **Дата**, «Состав даты» = **Дата** (без времени).

### 1.3 Документ `Ведомость`
Добавить документ, Имя = `Ведомость`.
- Вкладка **Нумерация**: «Тип номера» = **Строка**, «Длина номера» = **11**
  (чтобы влезло `БТ02-000016`); снять флажок **Автонумерация** — номера будем
  вводить руками, как в примере.
- Вкладка **Данные → Табличные части**: добавить ТЧ с именем `Договоры`.
  - Внутри неё **Реквизиты** → добавить `Договор`, тип **СправочникСсылка.Договоры**.

### 1.4 Регистр сведений `ИсторияСтатусовДокументов`
Добавить регистр сведений, Имя = `ИсторияСтатусовДокументов`.
- Вкладка **Основные**: «Режим записи» = **Независимый**; «Периодичность» = **Непериодический**.
- Вкладка **Данные → Измерения** (все три — измерения, как в задании):
  - `ДатаЗаписи` — тип **Дата**, «Состав даты» = **Дата и время**;
  - `Документ` — тип **составной**: в выборе типа включить «Составной тип данных»
    и отметить **ДокументСсылка.Ведомость**;
  - `Статус` — тип **ПеречислениеСсылка.СтатусыДокументов**.
- Ресурсы не нужны.

### 1.5 Подсистема (чтобы объекты были видны в режиме Предприятие)
Общие → Подсистемы → Добавить, Имя = `Данные`. На вкладке **Состав** отметить
Справочник `Договоры`, Документ `Ведомость`, Регистр `ИсторияСтатусовДокументов`.
(Без подсистемы в управляемом приложении не будет команд для ручного ввода данных.)

### 1.6 Обновить базу
**Конфигурация → Обновить конфигурацию базы данных** (F7). Принять изменения структуры.

## Фаза 2. Тестовые данные

Запустить **1С:Предприятие** (из Конфигуратора — F5). Ввести данные из раздела
«Пример данных» задания через панель разделов (подсистема «Данные»):

1. **Договоры** — 12 элементов (`Договор_1`…`Договор_12`) с их `Активен` и `НачалоДействия`.
2. **Ведомости** — 8 документов. У каждого задать **Номер** (напр. `БТ02-000016`) и
   **Дату** из примера, заполнить табличную часть `Договоры` в том же порядке строк.
3. **История статусов документов** — открыть регистр, «Создать» на каждую строку
   из примера: `ДатаЗаписи` (с временем!), `Документ` (выбрать Ведомость), `Статус`.

> Время в `ДатаЗаписи` важно: у `ЛР04-000256` две записи 11.04 отличаются на 1 секунду
> (10:04:26 vs 10:04:25) — именно это делает её последний статус «Заблокирован», а не
> «Акцептован». Вводите секунды точно.

> Ввод руками долгий и ошибкоопасный — могу вместо этого дать обработку
> «ЗаполнитьТестовыеДанные», которая создаст всё это одним нажатием (скажи).

## Фаза 3. Внешняя обработка

Конфигуратор → **Файл → Новый → Внешняя обработка**. Имя = `ОбработкаВедомостей`.

### 3.1 Основная форма
Ветка **Формы** → Добавить → «Форма» (управляемая), Имя = `Форма`, отметить как основную.

**Реквизиты формы** (правая панель редактора формы, вкладка Реквизиты):
- `ТаблицаВедомостей` — тип **ТаблицаЗначений**; добавить колонки (подчинённые реквизиты):
  | Колонка | Тип |
  |---|---|
  | `Ведомость` | ДокументСсылка.Ведомость |
  | `Номер` | Строка, длина 20 |
  | `ДатаВедомости` | Дата (Дата и время) |
  | `КоличествоДоговоров` | Число, длина 10 |
  | `ПоследнийСтатус` | ПеречислениеСсылка.СтатусыДокументов |
- `ВариантСравнения` — **Число** (длина 1). В свойстве «Список выбора» добавить:
  `0 → «равна»`, `1 → «не равна»`.
- `ВариантЭкстремума` — **Число** (длина 1). «Список выбора»:
  `0 → «минимальной»`, `1 → «максимальной»`.

**Команды формы** (вкладка Команды):
- `ЗаполнитьНужное`
- `УдалитьЛишнее`

**Элементы формы** (перетаскиванием на форму):
- Перетащить `ТаблицаВедомостей` на форму → появится таблица. У колонок
  `Номер`, `ДатаВедомости`, `КоличествоДоговоров`, `ПоследнийСтатус` снять
  свойство **Видимость** (на форме остаётся только `Ведомость`).
- Перетащить `ВариантСравнения` → у элемента «Вид» = **Поле переключателя**
  (радиокнопки «равна»/«не равна»). То же для `ВариантЭкстремума`.
- Перетащить команды `ЗаполнитьНужное` и `УдалитьЛишнее` в командную панель формы
  (или на форму как кнопки).

**Модуль:** открыть «Модуль формы», вставить весь код из
`ОбработкаВедомостей.Форма.МодульФормы.bsl`. Проверить, что имена обработчиков
команд (`ЗаполнитьНужное`, `УдалитьЛишнее`) совпали с командами (обычно 1С сама
связывает по имени; иначе в свойстве команды «Действие» указать одноимённую процедуру).

Условное оформление конструктором **не** трогаем — оно ставится из кода.

### 3.2 Форма `ФормаСообщения`
Формы → Добавить → «Форма», Имя = `ФормаСообщения`.
- **Параметры формы** (вкладка Параметры): `Текст` — **Строка**.
- **Реквизиты**: `ТекстСообщения` — **Строка**, «Неограниченная длина».
- **Элемент**: перетащить `ТекстСообщения` на форму → «Многострочный режим» = Да,
  «ТолькоПросмотр» = Да, растянуть.
- Свойство формы «Заголовок» = `Результат обработки`.
- Модуль: вставить код из `ОбработкаВедомостей.ФормаСообщения.МодульФормы.bsl`.

### 3.3 Сохранить
**Файл → Сохранить** → сохранить `ОбработкаВедомостей.epf` в удобную папку.
Это и есть сдаваемый файл.

## Фаза 4. Запуск и проверка

1. Режим **1С:Предприятие** → **Файл → Открыть** → выбрать `ОбработкаВедомостей.epf`.
2. Переключатели по умолчанию «равна» + «минимальной» → **Заполнить нужное**.
   Ожидаемо (Пример 1): `БТ02-000016, ЛР04-000256, БТ02-000032, ЛР04-001024, БТ02-000128`,
   строки `БТ02-000016` и `ЛР04-001024` — светло-зелёные.
3. **Удалить лишнее** → откроется форма с текстом
   `Удалено документов: 4 (БТ02-000016, БТ02-000032, ЛР04-000256, ЛР04-001024).`,
   в таблице останется `БТ02-000128`.
4. Переключить на «максимальной» → Заполнить нужное → останется одна `БТ02-000016`
   (Пример 2).

> Учти персональное условие (1): если запускаешь в день месяца, совпадающий с днём
> даты какой-то ведомости, она **намеренно** исключается — и вывод разойдётся с
> Примером 1 из PDF. В данных задания дни: 1, 2, 10, 12, 16, 20. Если сегодня один
> из них — соответствующая ведомость не попадёт в таблицу (это правильно). Чтобы
> сверить «чистую» логику с PDF, временно закомментируй блок с `ТекущийДень` в
> `ЗаполнитьТаблицуВедомостейНаСервере`.
